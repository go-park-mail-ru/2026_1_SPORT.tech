# 2026_1_SPORT.tech
Репозиторий команды 2026_1_SPORT.tech

## Правила (by @grv_nikita)
#### Правила разработки фичи
* Перед реализацией согласовать и задокументировать api между frontend и backend.
#### Правила написания кода
* Следуем go style guide (может найдем хороший, прикрепим ссылку).
* Избегаем дублирования, выносим в либы, переиспользуем.
* Длина функции не более 40 строк, вложенность не более 3.
* Все переменные называем осознанно, нельзя использовать однобуквенные.
* Не используем магические числа/строки.
#### Правила оформления PR
* Не более 250 строк кода в PR (не считая тестов, кодгена и т.д.).
* Каждая фича должна быть покрыта unit или функциональными тестами.
* Понятное описание задачи в тикете.
* Название ветки: `<Id тикета>-<oписание>`, пример: `SPORT-15-add-linter`
* Название PR: `<Id тикета>: <Описание>`, пример: `SPORT-15: Add linter`
* * Желательно делать рефакторинг и изменения в разных PR.
* * Желательно сразу покрывать фичу логами, метриками.
#### Правила прохождения ревью
* PR (оформленный по правилам выше) приносите в личку тг `@grv_nikita`.
* Смотрю в течение суток. После можете пинговать каждый час.
* Комментарии можно обсуждать, критиковать.
* NIT комментарии - на ваше усмотрение.
